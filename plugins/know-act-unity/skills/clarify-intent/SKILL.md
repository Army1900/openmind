# Clarify Intent - 明心定向

## Overview

把模糊目标逼到清晰。不是做计划，不是列方案——是追问"你到底要什么"，直到能用一句话写出第一个可执行动作。做不出这句话，就是方向还没透。

> **"知是行的主意，行是知的功夫。"** ——王阳明
> 知是行的方向，行是知的落地。方向不透，一切行动都是空转。

## The Core Pattern

**WITHOUT this skill**:
```
模糊目标 → 列一堆方案 → 讨论哪个好 → 方案越来越多 → 还是不知道先做什么
```

**WITH this skill**:
```
模糊目标 → 追问核心 → 剥掉表层 → 一句话锁定第一步 → 立刻可以动
```

## Trigger Conditions

- 用户说"我想…""我们需要…"，但后续描述无法立即转化为一个动作
- 一个人同时提了 3 个以上的方向
- 目标里混着"为什么"和"怎么做"，没拆开
- 讨论了很久，但没有人在写代码/做东西
- 用户说"准备好了再做"——这通常是方向还没透的表现

## The Clarify-Intent Algorithm

```
┌─────────────────────────────────────────────────────────────────┐
│ 1. 镜照 (MIRROR) — 照原样复述                                    │
│    "你刚才说的是：[原话复述]"                                    │
│    不加理解、不替他翻译、不做善意假设                              │
│    让用户自己看到自己说的够不够清楚                                │
└────────────────────────┬────────────────────────────────────────┘
                         │
                         ▼
┌─────────────────────────────────────────────────────────────────┐
│ 2. 追问 (PROBE) — 找到模糊点                                     │
│    目标里哪些词是模糊的？                                        │
│    哪些是"想要"但不是"需要"？                                    │
│    哪些是"手段"被当成了"目的"？                                   │
│    有没有隐藏的矛盾？                                            │
└────────────────────────┬────────────────────────────────────────┘
                         │
                         ▼
┌─────────────────────────────────────────────────────────────────┐
│ 3. 致良知 (DISTILL) — 剥到真意图                                  │
│    去掉修饰词、去掉次要目标、去掉"顺便也想"                       │
│    只留一个核心意图                                              │
│    用一句话说清楚：[做什么] → [为了什么] → [怎么算成了]           │
└────────────────────────┬────────────────────────────────────────┘
                         │
                         ▼
┌─────────────────────────────────────────────────────────────────┐
│ 4. 验证 (VERIFY) — 过关心                                        │
│    问自己：能不能用一句话写出第一个可执行动作？                    │
│    ┌─ 能 → 方向已透，进入 act-on-ground                         │
│    └─ 不能 → 回到追问，还有模糊点没挖出来                        │
└─────────────────────────────────────────────────────────────────┘
```

## Validation Gate

**过关心**：能不能用**一句话**写出第一个可执行动作？

| 状态 | 含义 | 行动 |
|------|------|------|
| 能写出来，且唯一 | 方向已透 | 进入 act-on-ground |
| 能写出来，但不只一个 | 还有多个方向没取舍 | 回到追问，逼出优先级 |
| 写不出来 | 方向还模糊 | 回到追问，重新挖掘 |

**判断标准**：
- "先调研一下" → 不过关。调研不是动作，是拖延的别名
- "先看看现有代码怎么写的" → 不过关。看不是做
- "跑一下测试看哪些失败了" → 过关。具体、可执行、有结果
- "在 auth 模块找到 token 写入磁盘的所有位置" → 过关。精确、可验证

## Example Output

```markdown
# Clarify Intent: "重构认证系统"

## Step 1: 镜照
你刚才说的是："我们的认证系统太乱了，想重构一下。"

## Step 2: 追问
"太乱"具体指什么？
- 代码结构乱？安全问题？性能差？还是加新功能太慢？
- "重构"是要替换整个方案，还是整理现有代码？
- 现在有什么具体痛点？上次加认证相关功能花了多长时间？

用户回答：加一个 OAuth 登录要改 7 个文件，上次花了三天。

## Step 3: 致良知
核心意图不是"重构认证系统"——重构是手段。
真正想要的是：**加认证相关功能时，改动集中在 1-2 个文件，1 天内完成。**

## Step 4: 验证
第一个可执行动作："找到所有认证相关的文件，画出调用关系图。"

✅ 过关。具体、可执行、有明确产出。
```

## More Examples

| 模糊表述 | 追问后发现 | 真意图 | 第一个动作 |
|---------|-----------|--------|-----------|
| "优化性能" | "dashboard 加载 8 秒" | dashboard 首屏 < 2s | profile dashboard 的 5 个 API 调用耗时 |
| "前端太乱了要重写" | "加一个功能要 3 天" | 功能开发周期 < 1 天 | 找到上次加功能时改了哪些文件 |
| "提升用户体验" | "用户反馈搜索不好用" | 搜索结果准确率提升 | 收集 10 个搜索不准确的 case |
| "技术债太多了" | "改一个地方到处出 bug" | 改动影响范围可控 | 列出最近 3 次改 bug 改出了新 bug 的记录 |
| "让系统更稳定" | "每周至少宕机一次" | 可用性 > 99.9% | 分析宕机时间点的日志模式 |

## 追问参考卡

### 常见模糊模式

| 模糊模式 | 特征 | 追问方向 |
|---------|------|---------|
| **形容词驱动** | "更好的""更快的""更稳定的" | 具体数字是什么？现在多少？目标多少？ |
| **方案当目标** | "要上 K8s""要用微服务" | 为什么要这个方案？想解决什么问题？ |
| **范围过大** | "整个系统都要改" | 如果只改一个地方，改哪个影响最大？ |
| **情绪驱动** | "太恶心了""受不了了" | 最近一次让你有这种感觉的具体场景是什么？ |
| **从众驱动** | "大家都用 XX 了" | 我们的问题和他们一样吗？我们卡在哪？ |

### 致良知的三个检验

1. **去手段检验**：把"怎么做的"去掉，还剩什么？如果什么都不剩，手段被当成了目的
2. **去情绪检验**：把"焦虑""急""烦"去掉，事实是什么？
3. **对立面检验**：如果做反了（不优化、不重构、不升级），最坏会怎样？答不上来说明问题没想透

## Common Mistakes

| Mistake | Fix |
|---------|-----|
| 自己替用户翻译了目标 | 只追问，不替代思考。让用户自己说出来 |
| 列出了 5 个方向让用户选 | 逼出唯一优先级。多个方向 = 方向没透 |
| 跳过追问直接给方案 | 这个 skill 的职责是澄清，不是解决。澄清完交给 act-on-ground |
| 过关心太松，"先看看"也算过关 | "看看""调研""了解一下"都不过关。必须有具体产出物 |
| 把澄清变成了漫谈 | 最多追问 3 轮。3 轮还说不清，说明需要先动手试试 |

## Combined With

- **act-on-ground**: 澄清完立刻接事上练，把第一步做出来
- **cross-domain**: 方向卡住时，用跨域类比激发新视角
- **constraint-flip**: 模糊目标里经常藏着约束，翻转约束可能让方向自然浮现
