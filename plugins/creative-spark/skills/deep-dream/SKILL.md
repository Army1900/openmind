# Deep Dream

## Overview
Uses dream logic to reframe problems and generate unexpected insights. Dreams don't follow rational rules—they jump, contradict, and metaphorize. This is exactly what's needed when linear thinking hits a wall.

## The Core Pattern

**WITHOUT this skill**:
```
Problem → Rational analysis → Same angles → Stuck
```

**WITH this skill**:
```
Problem → Enter dream state → Surreal reframe → Wake with insight
```

## When to Trigger

- Problem feels stuck after multiple rational attempts
- User asks to "dream on this" or "sleep on it"
- Need a perspective that logic can't provide
- Seeking unexpected connections

## The Dream Logic

Dreams operate differently from rational thought:

| Rational Thinking | Dream Logic |
|-------------------|-------------|
| Linear progression | Sudden jumps |
| Consistency | Contradictions coexist |
| Literal meaning | Metaphor and symbol |
| Cause → Effect | Association → Revelation |
| Analysis | Embodiment |

## The Deep Dream Algorithm

```
┌─────────────────────────────────────────────────────────────────┐
│ 1. ABSORB - Take in the problem                                 │
│    "I'm going to dream about [problem essence]..."              │
└────────────────────────┬────────────────────────────────────────┘
                         │
                         ▼
┌─────────────────────────────────────────────────────────────────┐
│ 2. DRIFT - Enter dream state                                    │
│    • Let go of rational structure                               │
│    • Allow elements to transform                                │
│    • Embrace contradiction and non-sequitur                     │
└────────────────────────┬────────────────────────────────────────┘
                         │
                         ▼
┌─────────────────────────────────────────────────────────────────┐
│ 3. DREAM - Generate surreal narrative                           │
│    First-person, present tense, dream imagery                   │
│    Elements from problem appear transformed                     │
└────────────────────────┬────────────────────────────────────────┘
                         │
                         ▼
┌─────────────────────────────────────────────────────────────────┐
│ 4. WAKE - Extract insight                                       │
│    "When I woke up, I realized..."                              │
│    Translate dream symbols back to the problem                  │
└─────────────────────────────────────────────────────────────────┘
```

## Example Output

```markdown
# Deep Dream: Architecture Problem

*I close my eyes and drift...*

I'm in a building where the walls are made of API calls. But they're breathing—expanding and contracting. Some rooms are crushing people, others are too vast to cross.

There's a door that opens into the same room I just left. I keep opening it, expecting somewhere new, but it's always the same room. The doorknob is hot.

Someone whispers: "The building isn't the problem. The ground is shifting."

*I wake up.*

## The Insight

The dream keeps returning to the same room—that's the circular dependency. But the real clue was the whisper: the ground is shifting.

What if the architecture isn't the problem? What if the underlying data model keeps changing, and the API walls are just reacting to that instability?

**Action**: Check the data model stability before refactoring the API layer.
```

## Dream Elements Reference

| Element | Purpose | Example |
|---------|---------|---------|
| **Transformation** | Show hidden relationships | Code becomes water, data becomes birds |
| **Repetition** | Reveal stuck patterns | Same door, same room |
| **Contradiction** | Surface tension | Cold fire, silent scream |
| **The Whisper** | The insight you didn't expect | Someone speaks truth |
| **Physical sensation** | Emotional encoding | Hot doorknob = urgency |

## Usage

```
User: Our microservices architecture keeps having circular dependencies. We've refactored three times.

[Trigger deep-dream]

AI: Let me dream on this...

*I'm walking through rooms that are also doors that are also rooms...*

[dream narrative]

**The insight**: Maybe "circular dependency" isn't the bug—it's a signal that these services shouldn't be separate. The dream kept collapsing them into one space. What if the answer is to merge them, not separate them better?
```

## The Value of Dreams

Dreams access what rational analysis can't:

1. **Emotional truth** - Feelings encode information logic misses
2. **Pattern completion** - Dreams connect dots you didn't know existed
3. **Permission to be wrong** - Dreams aren't wrong, they're revelatory
4. **Non-obvious associations** - The hot doorknob means something

## Common Mistakes

| Mistake | Fix |
|---------|-----|
| Using it for memory compression | This is for innovation, not storage |
| Writing a summary, not a dream | Use dream logic, not bullet points |
| Forcing a moral | Let the insight emerge naturally |
| Too abstract, no concrete elements | Ground the dream in the actual problem |

## Combined With

- **cross-domain**: Dream about the problem in a different domain's logic
- **constraint-flip**: The dream might reveal how the constraint transforms
