# OpenMind - 思维开拓工具箱

一个 Claude 插件市场，通过非传统思维方法突破 AI 的创造力边界。

## 理念

**AI 约束强但创新不足。** 这些插件刻意引入"非理性"元素，激发突破性思维。

## 结构

```
openmind/
├── .claude-plugin/
│   └── marketplace.json
├── plugins/
│   ├── creative-spark/           # 奇思妙想 - 创新突破
│   │   └── skills/
│   │       ├── cross-domain/     # 跨域类比
│   │       ├── constraint-flip/  # 约束反转
│   │       └── deep-dream/       # 深度梦境
│   │
│   └── poetic-spotlight/         # 诗意点睛 - 本质凝练
│       └── skills/
│           └── poetic-essence/   # 诗意本质
│
│   └── know-act-unity/           # 知行合一 - 目标达成
│       └── skills/
│           ├── clarify-intent/   # 明心定向
│           ├── act-on-ground/    # 事上练
│           └── reflect-anchor/   # 知行复盘
└── templates/
```

## 安装

```bash
claude plugin install https://github.com/openmind/openmind
```

## 插件一：creative-spark (奇思妙想)

**3个创新突破技能**

| 技能 | 触发条件 | 作用 |
|------|----------|------|
| **cross-domain** | 问题卡住 | 从自然/艺术/其他领域借用解决方案 |
| **constraint-flip** | "因为X所以不能" | 把限制转化为特性 |
| **deep-dream** | 需要洞察 | 用梦境逻辑重构问题 |

### cross-domain
```
用户：如何优化数据库查询？

[cross-domain 触发]

AI：自然界中，猪笼草不追逐猎物——它创造不可抗拒的陷阱等待。
如果数据也这样，不是被查询，而是"流向"需要它的地方？
这提示了一种"数据引力"模型...
```

### constraint-flip
```
用户：我们没有营销预算。

[constraint-flip 触发]

AI：正因为没有预算，你的产品必须令人惊叹。
每个用户都必须想告诉三个朋友。这不是限制——
这是有资金支持的竞争对手没有的质量门槛。
```

### deep-dream
```
用户：架构有循环依赖，重构了三次还是有问题。

[deep-dream 触发]

AI：让我做个梦...

*我在一栋建筑里，墙壁是 API 调用。有扇门打开后还是同一个房间。
有人低语："问题不在建筑，是地在动。"*

**洞察**：也许问题不在架构，而是底层数据模型不稳定。
先检查数据模型。
```

## 插件二：poetic-spotlight (诗意点睛)

**用中国古诗凝练项目本质**

| 技能 | 触发条件 | 作用 |
|------|----------|------|
| **poetic-essence** | 项目里程碑 | 引用/化用古诗，定调当前阶段 |

### 三种模式

| 模式 | 触发 | 动作 |
|------|------|------|
| **WRITE** | 里程碑完成 | 引用古诗（无完全契合则化用） |
| **READ** | "有什么诗能道出..." | 找匹配的诗 + 解读项目 |
| **INTERPRET** | 用户给诗求解读 | 深度品读 + 行动启示 |

### WRITE 模式（里程碑时）
```
[完成一个重要功能后]

AI：这个里程碑要不要用诗定个调？

用户：好啊

AI：**"山重水复疑无路，柳暗花明又一村"**
——陆游《游山西村》

**解读**：
"山重水复"——四个方向，四堵墙。
但"又一村"——不是找到原来的路，是进入新的世界。

**对项目的启示**：
四次改方向不是迷路——是你在用排除法。
现在的"重新开始"，带着四次排除后的清晰。
```

### READ 模式
```
用户：增长到了平台期，有什么古诗能解读一下？

[poetic-essence READ]

AI：**"不识庐山真面目，只缘身在此山中"**
——苏轼《题西林壁》

**解读**：
你卡住了，是因为你还在"山"里——现有的用户群、渠道、假设。

**对项目的启示**：
增长停滞不是"推不动"——是"走不出"。
找3个完全不同于现有用户的人聊聊，他们告诉你山的另一面。
```

### INTERPRET 模式
```
用户：团队寄语是"长风破浪会有时"，帮我品品

[poetic-essence INTERPRET]

AI：这是李白《行路难》——写于他很迷茫的时候。

**深层解**：
"会有时"不是"现在"，是"终会"。
这不是胜利宣言——是低谷期的信念。

**对团队的启示**：
真正的力量不是"破浪"，而是"等风来"时的准备。
与其急着破浪，不如先挂好帆。
```

## 插件三：know-act-unity (知行合一)

**3 个基于王阳明哲学的目标达成技能**

> "知而不行只是未知" —— 说了但没做，就是还不知道。

| 技能 | 触发条件 | 作用 |
|------|----------|------|
| **clarify-intent** | 目标模糊、方向不清 | 追问逼出真意图，一句话锁定第一步 |
| **act-on-ground** | 规划过度、迟迟不动 | 剥到最小可验证步骤，立刻执行 |
| **reflect-anchor** | 阶段完成或失败 | 直面错误假设，校准方向 |

### 循环流程

```
clarify-intent (明心定向) → act-on-ground (事上练) → reflect-anchor (知行复盘) → 循环
```

### clarify-intent
```
用户：认证系统太乱了，想重构一下。

[clarify-intent 触发]

AI："太乱"具体指什么？加一个认证功能要改几个文件？花多久？

用户：加 OAuth 要改 7 个文件，上次花了三天。

AI：真正想要的不是"重构认证系统"——重构是手段。
核心意图：加认证功能时，改动集中在 1-2 个文件，1 天内完成。

第一个动作："grep 出所有引用 auth 的文件，画出调用关系。"
✅ 过关——一句话、可执行、有产出物。
```

### act-on-ground
```
用户：[方向明确了，但还没动手]

[act-on-ground 触发]

AI：先别重构。最小一步：grep 所有 auth 引用。

结果：17 个文件引用了 auth（不是预期的 7 个）。
11 个间接依赖，其中 8 个穿透了封装。

认知变化：这不是重构问题，是架构问题。范围比感知的大 2.4 倍。

下一步：验证那 8 个穿透封装的依赖，改 auth 时会不会断。
```

### reflect-anchor
```
[认证系统重构做了 3 步之后]

[reflect-anchor 触发]

AI：原始目标："加认证功能，改动 1-2 个文件，1 天完成。"
现在走到哪了？

行动推翻的假设：
- 以为是重构问题 → 错，是架构问题
- 以为涉及 7 个文件 → 实际 17 个

我以为对但其实错的：
"我以为这是代码质量问题。其实是抽象边界问题——auth 的内部实现没有被封装。"

校准方向：先建抽象层，再重构。
```

|------|--------------|
